---
name: commit-message
description: "Write conventional commit messages following the project's commit convention"
---

When creating commits, follow the conventional commits format:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `refactor:` for code refactoring
- `test:` for test additions
- `chore:` for maintenance tasks

Keep the subject line under 72 characters. Use the body for details.
