# parity-fixtures — shared Py↔TS behavioral parity harness

Language-neutral behavioral fixtures executed by BOTH SDKs against the SAME
input trees, asserting byte-identical canonical JSON output
(`s-parity-behavioral-harness`, feature `f-ts-parity-composition-v2`).

## Layout

```
tests/parity-fixtures/
├── composition/            # Composition Engine V2 (resolveDocument et al.)
│   ├── cases/*.yaml         # hand-authored input cases (see schema below)
│   └── expected/*.json      # FROZEN canonical output — generated by PYTHON
├── port-surface-parity.json # per-port member manifest (Py↔TS names +
│                            # justified asymmetries) — s-dna-port-surface-parity.
│                            # Executors: packages/sdk-py/tests/
│                            # test_port_surface_parity.py (Protocol
│                            # introspection) + packages/sdk-ts/tests/
│                            # port-surface-parity.test.ts (keyof-bound
│                            # PORT_SURFACE manifest). Undocumented drift = red.
├── workspace-*/cases.json  # Model B tenancy pure policy (dna.tenancy ↔
│                            # src/tenancy). Each dir is a self-contained
│                            # cases.json with case families keyed by op,
│                            # read by a Py test + its TS twin:
│                            #   workspace-resolution/ — identity→workspace
│                            #     resolve (f-ws-resolution, F2)
│                            #   workspace-invite/     — invite/accept authz +
│                            #     anti-impersonation bind (f-ws-invites, F3)
│                            #   workspace-ownership/  — first-owner probe +
│                            #     revoke RBAC + last-owner guard
│                            #     (f-ws-owner-provision)
└── mif/memories/<name>/MEMORY.md  # Real MIF Memory examples lifted verbatim
                             # (or lightly extended, noted inline) from
                             # SPECIFICATION.md §16 — market-fidelity
                             # conformance for `mif-spec.dev/v1 · Memory`
                             # (s-mif-passthrough-kind), read by BOTH SDKs'
                             # GenericBundleReader/Writer directly (bundle
                             # storage, no scope/manifest ceremony needed):
                             #   packages/sdk-py/tests/test_mif_memory_kind.py
                             #   packages/sdk-ts/tests/mif-memory-kind.test.ts
```

## Executors (both read these SAME files)

- Python (the **semantic reference**):
  `packages/sdk-py/tests/test_composition_parity_fixtures.py`
- TypeScript (must match the frozen Python output):
  `packages/sdk-ts/tests/composition-parity-fixtures.test.ts`

Each executor materializes the case's `tree` as a real `.dna/`-style
filesystem tree in a tmp dir, builds a bare kernel over a filesystem source,
runs `compute_resolution_chain` / `resolve_document` (Py) ↔
`computeResolutionChain` / `resolveDocument` (TS) on `target`, and deep-compares
`{chain, resolved: <ResolvedDocument.serialize()>}` against
`expected/<case>.json`.

## Case schema (`cases/*.yaml`)

```yaml
id: 01-inherit-simple          # matches the file stem
title: one-line description
notes: |                        # optional — what semantics this case pins
  ...
skip_ts: "reason"               # optional — TS executor skips with this reason
target:
  scope: app
  kind: Theme
  name: brand
  tenant: null                  # or a tenant slug ("acme")
tree:
  scopes:                       # base-layer docs, per scope dir
    _lib: [ <raw doc>, ... ]
    app:  [ <raw doc>, ... ]
  tenants:                      # tenant overlay docs →
    acme:                       #   <base>/tenants/<t>/scopes/<s>/*.yaml
      app: [ <raw doc>, ... ]
```

A scope listed with an empty doc list still gets its directory created
(pins "scope exists but has no Genome" semantics).

## Freezing / regenerating `expected/`

Python is the semantic reference — the frozen output is ALWAYS produced by
the Python executor, never by hand and never by TS:

```bash
cd packages/sdk-py
DNA_REGEN_PARITY_FIXTURES=1 uv run pytest tests/test_composition_parity_fixtures.py -q
git diff tests/parity-fixtures/   # review EVERY change before committing
```

A TS failure against frozen expected is a TS bug (or an intentional,
documented semantic change that must land in Python FIRST and be re-frozen).
