# parity-fixtures ‚Äî shared Py‚ÜîTS behavioral parity harness

Language-neutral behavioral fixtures executed by BOTH SDKs against the SAME
input trees, asserting byte-identical canonical JSON output
(`s-parity-behavioral-harness`, feature `f-ts-parity-composition-v2`).

## Executors (both read these SAME files)

- Python (the **semantic reference**):
  `packages/sdk-py/tests/test_composition_parity_fixtures.py`
- TypeScript (must match the frozen Python output):
  `packages/sdk-ts/tests/composition-parity-fixtures.test.ts`

Each executor materializes the case's `tree` as a real `.dna/`-style
filesystem tree in a tmp dir, builds a bare kernel over a filesystem source,
runs `compute_resolution_chain` / `resolve_document` (Py) ‚Üî
`computeResolutionChain` / `resolveDocument` (TS) on `target`, and deep-compares
`{chain, resolved: <ResolvedDocument.serialize()>}` against
`expected/<case>.json`.

## Case schema (`cases/*.yaml`)

```yaml
id: 01-inherit-simple          # matches the file stem
title: one-line description
notes: |                        # optional ‚Äî what semantics this case pins
  ...
skip_ts: "reason"               # optional ‚Äî TS executor skips with this reason
target:
  scope: app
  kind: Theme
  name: brand
  tenant: null                  # or a tenant slug ("acme")
tree:
  scopes:                       # base-layer docs, per scope dir
    _lib: [ <raw doc>, ... ]
    app:  [ <raw doc>, ... ]
  tenants:                      # tenant overlay docs ‚Üí
    acme:                       #   <base>/tenants/<t>/scopes/<s>/*.yaml
      app: [ <raw doc>, ... ]
```

A scope listed with an empty doc list still gets its directory created
(pins "scope exists but has no Genome" semantics).

## Freezing / regenerating `expected/`

Python is the semantic reference ‚Äî the frozen output is ALWAYS produced by
the Python executor, never by hand and never by TS:

```bash
cd packages/sdk-py
DNA_REGEN_PARITY_FIXTURES=1 uv run pytest tests/test_composition_parity_fixtures.py -q
git diff tests/parity-fixtures/   # review EVERY change before committing
```

A TS failure against frozen expected is a TS bug (or an intentional,
documented semantic change that must land in Python FIRST and be re-frozen).
