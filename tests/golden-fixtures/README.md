# golden-fixtures — behavioral goldens for the DNA runtime

Language-neutral fixtures that **freeze behavior which crosses a boundary**:
a public API surface, a wire format, a resolution outcome, a security policy.
They are committed expected output — a code change cannot alter them by
accident, only by re-freezing them deliberately in the same PR.

> **History.** These began as a Py↔TS *parity* harness
> (`s-parity-behavioral-harness`, `f-ts-parity-composition-v2`): the same
> files were executed by the Python kernel and by its hand-maintained
> TypeScript twin. The twin was frozen at the tag `sdk-ts-final`. The
> fixtures stayed because their value never depended on it — Python was
> always the semantic reference, and what these files pin is Python behavior.

Package-local goldens — the MIF interchange projection, memory scoring, the
query/count core, `StudioUIMetadata`, and the emit artifacts — live next to
their suites in `packages/sdk-py/tests/goldens/`.

## The composition executor

`packages/sdk-py/tests/test_composition_golden.py` materializes each case's
`tree` as a real `.dna/`-style filesystem tree in a tmp dir, builds a bare
kernel over a filesystem source, runs `_compute_resolution_chain` /
`resolve_document` on `target`, and deep-compares
`{chain, resolved: <ResolvedDocument.serialize()>}` against
`expected/<case>.json`.

## Case schema (`cases/*.yaml`)

```yaml
id: 01-inherit-simple          # matches the file stem
title: one-line description
notes: |                        # optional — what semantics this case pins
  ...
target:
  scope: app
  kind: Theme
  name: brand
  tenant: null                  # or a tenant slug ("acme")
tree:
  scopes:                       # base-layer docs, per scope dir
    _lib: [ <raw doc>, ... ]
    app:  [ <raw doc>, ... ]
  tenants:                      # tenant overlay docs →
    acme:                       #   <base>/tenants/<t>/scopes/<s>/*.yaml
      app: [ <raw doc>, ... ]
```

A scope listed with an empty doc list still gets its directory created
(pins "scope exists but has no Genome" semantics).

## Freezing / regenerating `expected/`

The frozen output is ALWAYS produced by the executor, never by hand:

```bash
cd packages/sdk-py
DNA_REGEN_GOLDEN_FIXTURES=1 uv run pytest tests/test_composition_golden.py -q
git diff tests/golden-fixtures/   # review EVERY change before committing
```

A diff in `expected/` means resolution behavior changed. That is either a bug
you just caught, or an intentional semantic change — review it as such.
