Autorar um agente

03

Configurar a infraestrutura

Escolher os ports: "quero filesystem como source e postgres para o resto". Hoje não há caminho declarativo — nem o CLI wireia postgres; ele delega pro dev construir o adapter na mão.

Agora wiring imperativo, em código
from dna import Kernel
from dna.adapters.sqlalchemy_ import SqlAlchemySource
from dna.adapters.search import PgVectorProvider

# você precisa saber o nome de cada classe e como montar
source = SqlAlchemySource("postgresql://user@host/db")
search = PgVectorProvider("postgresql://user@host/db")
kernel = Kernel.auto(source=source)
kernel.search(search)
# Kernel.quick() nem oferece a escolha — é filesystem, ponto.
✦
declarativo
Depois dna.config.yaml
# dna.config.yaml — agnóstico de linguagem (Py + TS)
source:    postgresql://user@host/db   # ou file://./.dna
search:    pgvector                    # ou sqlite-vec / off
embedding: onnx                        # ou fake / off

# ─── e o consumidor: ───────────────────────────────
from dna import Kernel
mi = Kernel.from_config().manifest("helpdesk")

O paved road

O engine de composição, os ports, a paridade Py/TS — tudo continua. O que muda é que o caso 80% passa a ser trivial, e o poder fica reservado pros 20% que precisam. Os três quick wins abaixo, sozinhos, já viram a chave — e cabem num v0.4.0.

v0.4.0

build_prompt fail-loudAgente inexistente levanta AgentNotFound em vez de devolver a string "not found". Mata o guard mais perigoso do consumidor.

v0.4.0

saída de composição limpaSem newlines finais espúrios. O .rstrip("\n") some do lado de quem consome.

v0.4.0

dna.load_prompts(scope)Loader de alto nível: fail-loud embutido, saída limpa, resolução de base_dir. Colapsa as 166 linhas em 5.

v0.4.0

Kernel.from_config()Lê o dna.config.yaml e wireia source + search + embedding. O factory URL→adapter sai do CLI e vira API pública.

próx.

layout: persona-firstPresets de composição nomeados — o Mustache cru deixa de vazar no YAML de todo agente.
